# Структура модулей игры

## Описание структуры

Проект разделен на модули по принципам ООП для улучшения поддерживаемости и читаемости кода.

## Структура папок

### `/core` - Ядро игры
- **GameManager.js** - Главный координатор всех систем
- **GameConfig.js** - Конфигурация и константы
- **GameState.js** - Состояние игры
- **EventManager.js** - Управление событиями
- **ScoreManager.js** - Управление очками
- **SpawnManager.js** - Управление спавном
- **CameraManager.js** - Управление камерой

### `/entities` - Игровые сущности
- **Bullet.js** - Пули
- **Enemy.js** - Враги
- **Boss.js** - Боссы
- **DogEnemy.js** - Враг-собака
- **Particle.js** - Частицы
- **Building.js** - Здания
- **Wall.js** - Стены и укрытия
- **Trap.js** - Ловушки
- **PowerUp.js** - Пауэр-апы
- **Can.js** - Банка
- **Grenade.js** - Гранаты
- **Money.js** - Деньги
- **Garbage.js** - Мусор
- **Puddle.js** - Лужи
- **ZipLine.js** - Зиплайны
- **BgCar.js** - Фоновые машины

### `/ui` - Интерфейс
- **HUD.js** - Игровой интерфейс
- **Menu.js** - Главное меню
- **Store.js** - Магазин
- **PauseMenu.js** - Меню паузы
- **EndScreen.js** - Экран окончания
- **MeleeKillUI.js** - UI ближнего боя

### `/environment` - Окружение
- **Background.js** - Фон
- **Ground.js** - Земля/пол
- **EnvironmentManager.js** - Менеджер окружения

### `/physics` - Физика
- **PhysicsManager.js** - Управление физикой
- **CollisionDetector.js** - Обнаружение коллизий

### `/storage` - Хранилище
- **StorageManager.js** - Управление данными

### `/resources` - Ресурсы
- **ResourceLoader.js** - Загрузка ресурсов

### `/utils` - Утилиты
- **GameUtils.js** - Вспомогательные функции

## Порядок рефакторинга

1. **Этап 2** - Утилиты и конфигурация
2. **Этап 3** - Хранилище и ресурсы
3. **Этап 4** - Физика
4. **Этап 5** - Игровые сущности (по одной)
5. **Этап 6** - Окружение
6. **Этап 7** - UI
7. **Этап 8** - Менеджеры игры
8. **Этап 9** - Рефакторинг main.js
9. **Этап 10** - Тестирование

## Примечания

- Все модули должны экспортировать классы/функции для использования в других модулях
- Избегать циклических зависимостей
- Сохранять работоспособность на каждом этапе
- Тестировать после каждого этапа
